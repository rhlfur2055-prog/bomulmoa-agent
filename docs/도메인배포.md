# bomulmoa.com 도메인 배포 가이드 (Vultr 직접 서빙)

Treasure-Collective 레포 접근이 불가능하므로, 우리 서버(**bomulmoa-bot, 158.247.244.88**, Ubuntu 24.04)에서
nginx로 `site/`를 직접 서빙하고 Cloudflare DNS만 이 서버로 돌리는 방식입니다.

매일 08:40 cron(`publish-site.sh`)이 `site/`를 재생성하므로, 한 번 연결해두면 새 글·시세가 자동으로 라이브에 반영됩니다.

---

## 0단계: 전제 조건

1. **main 머지 후 서버에서 최신 코드 받기**

   ```bash
   ssh root@158.247.244.88
   cd /opt/bomulmoa   # repo 위치
   git pull
   ```

2. **site/data.js 실전화 필수**
   - 연락처(전화번호·카카오톡 등)와 **실주소**가 진짜 값인지 반드시 확인하세요.
   - ⚠️ **가짜 전화번호/주소가 들어간 상태로 라이브에 올리면 안 됩니다.** 고객이 실제로 전화하는 번호입니다.
   - 단가도 최신인지 확인 (`npm run export-prices` 로 갱신 가능).

## 1단계: 서버에서 nginx 서빙 시작

```bash
cd /opt/bomulmoa
bash deploy/serve-site.sh
```

확인:

```bash

# → HTTP/1.1 200 OK 가 나오면 정상
```

브라우저에서 **http://158.247.244.88/** 접속 → 보물모아 사이트가 보이면 서버 준비 완료.

여러 번 실행해도 안전하니, 설정을 바꾸고 싶으면 스크립트 수정 후 다시 실행하면 됩니다.

## 2단계: Cloudflare에서 도메인 연결 (택1)

Cloudflare 대시보드 → **bomulmoa.com** 도메인 선택 후, 아래 두 옵션 중 하나를 선택합니다.

### 옵션 A (권장, 간단): Cloudflare 프록시 + Flexible SSL

1. **DNS** 메뉴에서:
   - `bomulmoa.com` **A 레코드** → `158.247.244.88` 로 변경
   - `www` **A 레코드**도 동일하게 `158.247.244.88` 로 변경 (없으면 새로 추가)
   - 두 레코드 모두 **프록시 켜짐(주황 구름)** 유지
2. **SSL/TLS** 메뉴 → 암호화 모드를 **"Flexible"** 로 설정
   - (서버는 80 포트 HTTP만 열려 있고, 방문자↔Cloudflare 구간은 HTTPS로 처리됨)
3. **⚠️ Workers Routes에서 `bomulmoa.com/*` 라우트 제거** — 가장 중요!
   - **Workers Routes** (또는 Workers & Pages → 해당 Worker → Settings → Triggers/Routes)에서
     `bomulmoa.com/*`, `www.bomulmoa.com/*` 같은 라우트를 **삭제**하세요.
   - 기존 Workers 배포(옛 사이트)가 요청을 가로채기 때문에, **이걸 안 지우면 A 레코드를 바꿔도 계속 옛 사이트가 뜹니다.**

### 옵션 B (정석): DNS only + 서버에서 HTTPS 직접 발급

1. **DNS** 메뉴에서 A 레코드(`bomulmoa.com`, `www`)를 `158.247.244.88` 로 변경하되,
   **프록시 꺼짐(회색 구름, DNS only)** 으로 설정
2. 옵션 A와 동일하게 **Workers 라우트 제거**
3. 서버에서 certbot으로 인증서 발급:

   ```bash
   # certbot 설치 (snap 경로, Ubuntu 24.04 권장 방식)
   snap install --classic certbot
   ln -sf /snap/bin/certbot /usr/bin/certbot

   # HTTPS 인증서 발급 + nginx 자동 설정
   certbot --nginx -d bomulmoa.com -d www.bomulmoa.com
   ```

   - 이메일 입력·약관 동의 후 자동으로 nginx에 HTTPS 설정이 추가됩니다.
   - 갱신은 certbot이 자동 처리(타이머 내장).

## 3단계: 최종 확인

1. 브라우저에서 **https://bomulmoa.com** 접속 → **새 사이트**(보물모아 시세·소식)가 보이는지 확인
2. `www.bomulmoa.com` 도 확인
3. 옛 사이트가 계속 보이면:
   - Cloudflare **Caching → Configuration → Purge Everything** 으로 캐시 퍼지
   - Workers 라우트가 정말 삭제됐는지 재확인 (옵션 A의 3번)
   - 브라우저 강력 새로고침(Ctrl+Shift+R)

## 4단계: 이후 자동 운영

- **매일 08:40** cron의 `publish-site.sh`가 `site/`(시세·소식 페이지)를 재생성합니다.
  nginx는 그 디렉터리를 그대로 서빙하므로 **별도 배포 없이 새 글이 곧바로 라이브에 반영**됩니다.
- 기존 Treasure-Collective 레포 푸시 단계(`TREASURE_DIR`)는 **더 이상 필요 없습니다.**
- 단가를 바꾸고 싶으면 시트 수정 후 서버에서:

  ```bash
  cd /opt/bomulmoa && npm run export-prices
  ```

  만 실행하면 됩니다 (다음 방문자부터 새 단가 표시).

## 롤백 (문제 생겼을 때)

서버 쪽은 건드릴 필요 없이 **Cloudflare에서만 원복**하면 즉시 이전 상태로 돌아갑니다.

1. DNS A 레코드를 원래 값으로 되돌리기
2. 삭제했던 Workers 라우트(`bomulmoa.com/*`)를 다시 추가
3. 캐시 퍼지(Purge Everything)

DNS/라우트 변경은 Cloudflare 프록시 특성상 보통 수 분 내 반영됩니다.
