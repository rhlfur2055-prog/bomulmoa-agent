# 로컬 검색 장악 플레이북 (사장님 실행용)

> 이 사이트가 하는 일은 딱 둘. **(a) 검색에 잡히는 것, (b) 전화 오게 하는 것.**
> 아래는 그 둘을 위해 사장님이 직접 손으로 하는 체크리스트다. 상상 아니고, 오늘 할 수 있는 일만 적었다.

---

## ★ 시작 전 전제 (이거 안 하면 아래 전부 무의미)

- [ ] **★ `site/data.js`에 실제 정보 입력** — 3개 지점 실전화번호·실주소·영업시간. 지금은 `0507-0000-000X` 같은 가짜 번호라 검색용 사업자정보(JSON-LD)가 아예 안 나간다. 진짜 번호를 넣어야 구글·검색이 "이 가게 진짜다"라고 인식한다.
- [ ] **★ 좌표(lat/lng) 입력** — 네이버 지도에서 주소 검색 → 정확한 위치의 위도·경도를 `data.js`에 넣는다. (지도 노출·거리순 검색에 쓰인다)
- [ ] **★ 사이트 배포** — `bomulmoa.com` 도메인으로 실제 접속되게 올린다. 배포 안 하면 검색엔진이 색인할 대상이 없다.
- [ ] 위 3개 중 하나라도 안 됐으면, 아래 네이버 플레이스 등록부터 먼저 해도 된다(플레이스는 사이트와 독립).

---

## 1. 네이버 스마트플레이스 등록 (제일 중요 — 여기부터)

한국에서 고물상 찾는 사람은 99% **네이버 지도/플레이스**로 검색한다. 3개 지점을 **각각 따로** 등록한다.

### 지점별 등록 절차 (대림자원·모두다자원·대성자원 각 1회씩, 총 3번)

1. `smartplace.naver.com` 접속 → 네이버 아이디 로그인 → **[신규 등록]**.
2. **업종 선택**: `고물상` 또는 `재활용품수집` / `고철·비철금속 매입`. (검색되는 대로 가장 가까운 것. "고물상"이 1순위)
3. **업체명(상호)**: `data.js`·간판·사업자등록증과 **글자 하나까지 똑같이**. 예) `대림자원`. 지점 구분이 필요하면 `대림자원 (부평)`처럼 통일된 형식으로.
4. **주소**: 도로명주소 정확히. 지도 위 핀을 실제 야적장 입구로 옮긴다.
5. **전화번호**: `data.js`와 동일한 대표번호. 휴대폰이면 휴대폰, 070이면 070으로 통일.
6. **영업시간**: 평일 08:00–18:00 / 토 08:00–13:00 등 실제대로. 명절·일요일 휴무 표시.
7. **대표키워드(찾아오는 검색어)**: 지역+품목 조합으로 채운다. 예)
   - 대림자원(부평): `부평 고물상`, `부평 고철 매입`, `부평 비철`, `부평 방문수거`, `부평공단 고철`, `청천동 고물상`
   - 대성자원(부천): `부천 고물상`, `부천 고철 매입`, `부천 비철`, `오정동 고물상`, `부천 공장 스크랩`
   - 모두다자원(인천): `인천 고물상`, `인천 고철 매입`, `남동공단 고철`, `인천 비철`, `주안 고물상`
8. **사진(필수, 많을수록 좋음)** — 아래 4종은 꼭 올린다:
   - **간판** (상호 잘 보이게, 낮에)
   - **계근대** (트럭 올라간 사진이면 신뢰도↑ — "눈앞에서 계근" 강조)
   - **야적장** (물량 쌓인 모습 = 진짜 영업 중이라는 증거)
   - **수거 차량** (트럭/집게차 — "방문수거 진짜 온다")
   - 추가로 구리·알루미늄 등 품목 사진, 정산 장면.
9. **소개글**: 사이트 지역 페이지 문구 재활용. "고철·비철·폐지, 오늘 단가로 매입. 소량 환영, 방문수거, 현장 계근 즉시 정산."
10. **홈페이지 링크**: 각 지점을 담당 지역 랜딩페이지로 연결.
    - 대림자원 → `bomulmoa.com/region/bupyeong.html`
    - 대성자원 → `bomulmoa.com/region/bucheon.html`
    - 모두다자원 → `bomulmoa.com/region/incheon.html`
11. **소유확인(전화 인증)**: 등록한 대표번호로 오는 ARS/문자 인증 완료. 이거 해야 정식 노출.

### 리뷰 요청 (노출 순위에 직접 영향)

- 거래 끝나고 정산해줄 때, **단골·좋게 거래한 손님한테만** "네이버에 방문자리뷰 한 줄 남겨주시면 큰 힘 됩니다" 부탁.
- 계근표/명함에 **네이버 플레이스 QR코드**를 붙여 그 자리에서 남기게 한다.
- 별점 구걸·대가 지급 금지(어뷰징으로 노출 깎임). 자연스러운 실거래 리뷰가 제일 세다.
- 리뷰 달리면 **반드시 답글**. "감사합니다, 또 필요하시면 전화 주세요 010-XXXX" — 답글에도 지역·전화 들어가면 검색에 도움.

## 2. 다른 지도·검색 등록 (플레이스 다음 우선순위)

- [ ] **네이버 지도**: 스마트플레이스 등록하면 자동 연동. 지도앱에서 상호 검색해 실제로 뜨는지 확인.
- [ ] **카카오맵 (카카오 비즈니스)**: `business.kakao.com` → 카카오맵 장소 등록. 네이버와 **동일한 상호·주소·전화**로. 카톡으로 문의 오는 손님 대비.
- [ ] **구글 비즈니스 프로필**: `business.google.com` → 3개 지점 등록 → 우편/전화 인증. 구글 지도·구글 검색 노출용. 사진·영업시간·홈페이지 링크 동일하게.
- [ ] 등록 후 각 플랫폼에서 나온 **플레이스 링크를 `data.js`의 `naverPlaceUrl`에 넣으면** 사이트 거점 카드·지역 페이지에 "네이버 지도" 버튼이 자동으로 생긴다.

## 3. 로컬 SEO 기본기 (한 번 맞춰두면 계속 먹는다)

- **NAP 일관성이 전부다.** NAP = 상호(Name)·주소(Address)·전화(Phone).
  - `data.js`, 네이버 플레이스, 카카오맵, 구글, 명함, 간판, 세금계산서 — **전부 똑같은 글자**로.
  - "대림자원" / "대림 자원" / "대림자원(주)" 이렇게 제각각이면 검색엔진이 다른 가게로 본다. **하나로 통일.**
- **지역+품목 키워드로 생각한다.** 사람들은 "고철 팔기"가 아니라 "**부평 고철 매입**", "**부천 고물상**"으로 검색한다. 상호만 알리려 하지 말고 지역명을 항상 붙인다.
- **이 사이트가 검색에 기여하는 방식:**
  - **지역 랜딩페이지 3장**(`region/bupyeong·bucheon·incheon.html`) — 각 페이지가 "부평 고철 매입", "부천 비철 방문수거" 같은 지역 검색어에 정확히 대응하는 별도 URL이라, 검색엔진이 지역별로 각각 색인한다. 홈 한 장만 있을 때보다 잡히는 검색어가 넓어진다.
  - **JSON-LD 사업자정보** — 각 페이지가 지점 상호·전화·주소·영업시간을 검색엔진이 읽는 구조화 데이터로 자동 송출(★실번호 입력 시). 구글이 "지역 업체"로 인식하는 데 쓰인다.
  - **자동 소식·시세글**(`posts/`) — 시세 브리핑이 새 글로 계속 쌓이면 "오늘 고철 시세" 류 검색 유입이 생기고, 사이트가 죽지 않고 갱신된다는 신호(검색엔진이 좋아함)를 준다.
  - **플레이스 ↔ 사이트 상호 링크** — 플레이스에 사이트 주소, 사이트에 플레이스 링크를 서로 걸면 신뢰도가 올라간다.

## 4. 오프라인 발견 (온라인만큼 중요 — 전화는 여기서도 온다)

### 현수막·명함·차량 랩핑 문구 (지역+품목+전화, 이 3개 필수)

- 현수막(공단 진입로·거점 앞): **"부평 고철·비철 삽니다 · 방문수거 · 즉시 현금정산 ☎ 010-XXXX-XXXX"**
- 명함/스티커: 앞면 상호·전화 크게 / 뒷면 취급품목(고철·구리·알루미늄·폐지)·영업시간·**네이버 플레이스 QR**.
- 트럭 랩핑: 양옆에 **"고철·비철 방문수거 | 부평·부천·인천 | ☎ 010-XXXX"**. 트럭이 동네 돌 때가 광고다.
- 공통 원칙: **어느 지역 어떤 품목을 사는지 + 전화번호**가 3초 안에 읽혀야 한다. 예쁜 것보다 큰 글씨·큰 번호.

### 공장·상가 정기수거 영업 (부평·부천 공단이 핵심 타깃)

- **부평공단**(청천동·갈산동·삼산동), **부천 오정산업단지·테크노파크** — 금속가공·설비 공장은 매달 스크랩이 나온다. 정기수거처 1곳 = 매달 반복 물량.
- 명함 돌리며 "**공정 스크랩·자투리 자재 매달 정기수거 갑니다, 물량 있으시면 전화 주세요**" 한 줄.
- 단, **NOTES 원칙**대로: 영업 돌아 억지로 만들지 말고, **먼저 온 전화 중 같은 공장에서 두 번째 전화가 오면** 그때 "매달 이맘때 그냥 갈까요?" 한마디. 되는 쪽으로.

### "어디 보고 오셨어요?" — 유입경로 기록 습관 (NOTES 핵심)

- **모르는 번호로 전화 오면 딱 한 마디**: "어디 보고 전화 주셨어요?" → 수첩/휴대폰 메모에 한 줄.
  - 예) `7/12 부평 구리 문의 — 네이버 검색` / `7/13 부천 공장 철거 — 현수막 보고`
- 이게 **첫 데이터**다. 어디서 전화가 오는지 알아야 뭘 더 할지 정한다(네이버가 되면 리뷰·사진 더, 현수막이 되면 현수막 더).

## 5. 측정 — 되는지 안 되는지 딱 한 달 뒤 판정

- **세는 법(간단하게):** 달력이나 수첩에 **모르는 번호 전화가 온 날 정(正)자 하나**. 아는 거래처·스팸 빼고 "검색·현수막 보고 처음 연락한 사람"만.
- **주간 집계:** 매주 일요일에 그 주 정자 개수를 적는다. `1주차 0통 / 2주차 1통 / 3주차 ...`
- **판정 기준 (NOTES.md 인용):**
  > **"사이트 올리고 플레이스 등록했는데, 모르는 번호 전화가 안 온다 (한 달) → 온라인은 접는다. 사이트는 놔두되 시간 안 쓴다. 끝."**
  > **"모르는 번호 전화가 일주일에 2~3통 온다 → 그때 처음으로 어디 보고 오셨어요 물어서 수첩에 적는다. 이게 첫 데이터다."**
- 즉: **한 달 뒤 주 2~3통 이상이면 계속, 거의 0이면 접는다.** 감으로 정하지 말고 정자 개수로 정한다.
- 되면 다음 신호를 기다린다(NOTES): 같은 공장 두 번째 전화 → 정기수거 제안. 그 이상은 아직 상상이니 손대지 않는다.

### 한 장 요약

1. ★ `data.js` 실번호·실주소·좌표 넣고 사이트 배포.
2. 네이버 스마트플레이스 3곳 등록(업종=고물상, 사진 4종, 지역+품목 키워드, 전화인증).
3. 카카오맵·구글 비즈니스도 **같은 NAP**로 등록.

5. 모르는 번호 오면 "어디 보고 오셨어요?" 기록. 한 달 뒤 주 2~3통이면 계속, 아니면 접는다.
