## 2. 자동으로 돌아가는 것 — 손 안 대도 되는 부분

서버에 설치된 cron 5종 (CRON_TZ=Asia/Seoul, `deploy/crontab.example`):

| 주기 | 작업 | 하는 일 |
|---|---|---|
| 매일 08:00 | crawl-grants | 기업마당·K-Startup·구글뉴스 → 지원사업 시트 갱신 + 마감 임박 Slack 알림 |
| 매일 09:00 | watch-grants | 예비창업 공고 감시망 |
| 2시간마다 | watch-modoo | 「모두의 창업」 2차 공고 감시 |
| 6시간마다 | research | 고철·비철 시세·뉴스 리서치 → 시트 적재 |
| 매일 08:40 | publish-site | 시세(prices.js)·소식글(posts)·sitemap 생성 → Treasure-Collective 푸시 → 배포 |

**자동/수동의 경계 — 여기부터는 사람 일이다:**

- **자동**: 시세 수집, 소식글 생성, 사이트 발행, 공고 감시. 켜 두면 알아서 돈다.
- **수동**: 전화 받기, "어디 보고 오셨어요?" 묻고 정자 긋기, 플레이스 리뷰 답글, 단가 시트에 실단가 입력, 정기수거 제안 한마디. **전화가 오는지 세는 일은 어떤 cron도 대신 못 한다.**

## 3. 분기 시나리오 — 한 달 뒤 셋 중 하나

### 시나리오 A — 한 달간 모르는 번호 전화 0통

NOTES.md 그대로: **온라인은 접는다. 사이트는 놔두되 시간 안 쓴다. 끝.**

- **남는 것**: 인프라(사이트·자동발행은 그대로 돌아간다), 예창패 증빙(실행했다는 기록 자체가 증빙), 부지조사 문서(docs/부지조사-부평.md).
- **잃는 것**: 서버 비용 월 $6뿐 (Vultr vc2-1c-1gb). 그마저 아까우면 서버를 내리면 된다.
- 하지 말 것: "조금만 더 해보면"이라며 기능 추가. 전화 안 오는데 기능 추가 ❌ (NOTES).

### 시나리오 B — 모르는 번호 전화가 주 2~3통

- 그때 처음으로 "어디 보고 오셨어요?" 물어서 수첩에 적는다. **이게 첫 데이터다.**
- 유입경로별로 쏠리는 쪽을 더 한다 (네이버면 리뷰·사진, 현수막이면 현수막).
- **같은 곳에서 두 번째 전화가 오면** → "매달 이맘때 그냥 갈까요?" 한마디. 정기수거는 이렇게 하나씩 생긴다.
- **기능 추가 금지 원칙 유지**: 전화가 온다고 대시보드·장부·앱을 만들지 않는다. 거래가 쌓여서 수첩이 헷갈리기 시작할 때까지는 코드 금지.

### 시나리오 C — 정기수거처가 10곳을 넘는다

수첩이 헷갈리기 시작하는 지점. 여기서부터 비로소 다음 단계다.

1. **모아장부 WIP 재개** — 중단해 둔 Phase 2(Next.js 스캐폴드)부터. 필요가 확인됐으니 이제 만들어도 된다.
2. **납품처 격상 검토** — 물량이 명함이다. 월 취급량을 들고 위 단계 납품처에 전화를 돌려본다 (ROADMAP Stage 3의 예비 동작).
3. **부평 후보지 실사** — 삼산동(1순위) → 청천동(2순위) → 갈산동(3순위) 순서로 (docs/부지조사-부평.md). 정비구역·300m 주거 이격부터 확인.
4. **예창패 실증 확보** — 정기수거처 목록·월 물량 기록이 사업계획서의 실증 데이터가 된다.

## 4. 예창패 연결 — 어느 분기든 제출은 된다

사업계획서 초안은 이미 있다 (docs/예창패-사업계획서-초안.md). 공고는 cron(watch-grants·watch-modoo)이 잡는다.

| 분기 | 제출 가능? | 각도 |
|---|---|---|
| A (0통) | 가능 | "실행하고 측정했다"는 기록 자체가 증빙. 온라인 유입 가설을 검증하고 접은 판단 근거를 그대로 쓴다 |
| B (주 2~3통) | 가능 | 유입경로 데이터(첫 데이터)가 시장 검증 자료가 된다 |
| C (정기 10곳+) | 가능 · 최상 | 정기수거 실증 + 물량 기록 — 심사에서 가장 센 형태 |

즉 예창패 때문에 분기 판정을 미루거나 비틀 이유가 없다. 판정은 판정대로, 제출은 제출대로.

| 주기 | 보는 것 | 판정 기준 | 행동 |
|---|---|---|---|
| **주 1회** (일요일) | 그 주 모르는 번호 전화 정(正)자 수 | — (집계만) | 수첩에 주차별 통수 기록. 주 2~3통 도달 시 유입경로 묻기 시작 |
